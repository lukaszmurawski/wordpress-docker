# WordPress Docker — local development environment

A ready-to-use WordPress stack running on Docker with nginx, MySQL, PHP-FPM and WP-CLI. Supports HTTPS on a local domain with a self-signed SSL certificate.

## Stack

- **WordPress** `fpm-alpine` — PHP-FPM
- **Nginx** `alpine` — web server with SSL
- **MySQL** `8.0` — database
- **WP-CLI** — WordPress installation and management

---

## Requirements

- Docker Engine `20.10+`
- Docker Compose plugin (`docker compose`) or `docker-compose` v1
- `openssl` — for SSL certificate generation
- Administrator privileges — to edit the `hosts` file

---

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/lukaszmurawski/wordpress-docker.git
cd wordpress-docker
```

### 2. Run the install script

```bash
bash install.sh
```

The script will prompt for:

| Field | Description | Default |
|---|---|---|
| Domain | Local domain, e.g. `wordpress.local` | — |
| Site Title | WordPress site title | — |
| Admin User | Admin username | `admin` |
| Admin Pass | Admin password | *autogenerated* |
| Admin Email | Admin email | `admin@<domain>` |

The script automatically:
- generates random database passwords
- creates the `.env` file from `.env.dist`
- generates an SSL certificate for the given domain
- starts all containers
- installs WordPress via WP-CLI

## After installation

### 1. Update your hosts file

Map the local domain to `127.0.0.1`:

**Linux / macOS** — edit `/etc/hosts`:
```bash
sudo nano /etc/hosts
```

**Windows** — edit `C:\Windows\System32\drivers\etc\hosts` as administrator.

Add the following line (replace with your domain):
```
127.0.0.1   wordpress.local
```

### 2. Import the root certificate into your browser

The root certificate is located at `nginx/ssl/rootCA.pem`. Without importing it, your browser will show an untrusted certificate warning.

**Chrome / Edge:**
1. Go to `Settings` → `Privacy and security` → `Security`
2. Scroll to `Manage certificates` → `Trusted Root Certification Authorities`
3. Click `Import` and select `nginx/ssl/rootCA.pem`

**Firefox:**
1. Go to `Settings` → `Privacy & Security`
2. Scroll to `Certificates` → `View Certificates`
3. Open the `Authorities` tab → `Import` → select `nginx/ssl/rootCA.pem`
4. Check *This certificate can identify websites*

**macOS (system keychain):**
```bash
sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain nginx/ssl/rootCA.pem
```

## Starting containers after installation

Once installed, containers run in the background. To manage them later:

```bash
# Start
docker compose up -d

# Stop
docker compose down

# Check status
docker compose ps

# View logs
docker compose logs -f
```

---

## WP-CLI

The `wpcli` container uses the `cli` profile — it does not start automatically and is invoked on demand:

```bash
docker compose run --rm wpcli wp plugin list
docker compose run --rm wpcli wp user list
docker compose run --rm wpcli wp cache flush
```

## Environment variables

The `.env` file is generated automatically by `install.sh` from `.env.dist`.

| Variable | Description |
|---|---|
| `DOMAIN_NAME` | Local domain name |
| `DB_NAME` | Database name |
| `DB_USER` | Database user |
| `DB_PASSWORD` | Database user password (autogenerated) |
| `DB_ROOT_PASSWORD` | MySQL root password (autogenerated) |

## Using this as a base for your project
 
Feel free to use this repository as a starting point for your own WordPress projects. You can fork it, clone it, or simply copy the files — no attribution required.
 
Typical things you may want to customize:
- nginx configuration in `nginx/conf.d/default.conf.dist`
- PHP memory limit in `install.sh`
- MySQL version in `docker-compose.yaml`

## Note on the `wordpress/` directory
 
The `wordpress/` directory is intentionally excluded from version control (see `.gitignore`). This means **Git will not track any changes made inside it** — including plugins, themes, uploads, and `wp-config.php`.
 
If you want to version-control your theme or plugins, add them to Git explicitly or set up a separate repository for them.
